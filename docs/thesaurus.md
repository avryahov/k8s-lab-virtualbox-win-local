# Тезаурус / Thesaurus

Этот файл нужен как двуязычный учебный словарь проекта.

Он помогает:

- читать комментарии в скриптах;
- понимать документацию Kubernetes;
- сопоставлять русский термин с английским;
- не путаться в базовых словах при работе со `stage1`.

---

## Как пользоваться

Читай так:

- сначала русский термин;
- потом английский эквивалент;
- потом короткое объяснение простыми словами;
- затем, если нужно, смотри привязку к нашему проекту.

## Базовые термины проекта

| Русский термин | English term | Что это значит |
|---|---|---|
| виртуальная машина | virtual machine, VM | Отдельный компьютер внутри твоего компьютера. |
| хост | host | Основная машина, на которой всё запускается. В нашем случае это Windows. |
| гость | guest | Операционная система внутри виртуальной машины. В нашем случае это Ubuntu. |
| провижининг | provisioning | Автоматическая настройка машины скриптами после её создания. |
| сценарий | scenario | Последовательность шагов, которая выполняет конкретную задачу. |
| учебный стенд | training lab | Специально подготовленная среда для практики и повторения. |
| хвосты | leftovers, tails, artifacts | Остатки от прошлого запуска: файлы, токены, временные каталоги, orphan-VM. |
| чистый запуск | clean run | Запуск после полного удаления старого состояния и временного мусора. |

## Термины Kubernetes

| Русский термин | English term | Что это значит |
|---|---|---|
| кластер | cluster | Несколько нод, работающих вместе как единая система Kubernetes. |
| нода | node | Машина, которая является частью кластера. |
| master | control plane node, master | Управляющая нода, где работает API и основные управляющие компоненты. |
| worker | worker node | Нода, на которой обычно запускаются пользовательские Pod-ы. |
| API сервер | API server | Главная точка входа для команд `kubectl` и внутренних запросов кластера. |
| планировщик | scheduler | Компонент, который решает, на какой ноде запускать Pod. |
| kubelet | kubelet | Агент ноды, который следит за Pod-ами на этой машине. |
| kube-proxy | kube-proxy | Компонент, который помогает сервисной сети доставлять трафик до Pod-ов. |
| etcd | etcd | Хранилище состояния кластера. |
| namespace | namespace | Логическая область внутри кластера для разделения ресурсов. |
| манифест | manifest | YAML-файл с описанием Kubernetes-ресурсов. |
| ресурс | resource | Любой объект Kubernetes: Pod, Service, Deployment, Job и так далее. |

## Термины запуска кластера

| Русский термин | English term | Что это значит |
|---|---|---|
| bootstrap | bootstrap | Начальный этап, когда кластер только создаётся и начинает работать. |
| инициализация кластера | cluster initialization | Создание control plane через `kubeadm init`. |
| присоединение ноды | node join | Подключение worker-ноды к кластеру через `kubeadm join`. |

| kubeconfig | kubeconfig | Файл с параметрами доступа к кластеру Kubernetes. |
| idempotent | idempotent | Такой сценарий, который можно запускать повторно без поломки результата. |
| post-bootstrap | post-bootstrap | Шаги, которые выполняются уже после базового поднятия кластера. |
| финализация | finalization | Завершающая часть настройки после основного bootstrap. |

## Термины сети и CNI

| Русский термин | English term | Что это значит |
|---|---|---|
| Pod-сеть | Pod network | Сеть, через которую Pod-ы общаются между собой. |
| сервисная сеть | Service network | Внутренняя логическая сеть Kubernetes для Service-ов. |
| CNI | Container Network Interface | Механизм, через который Kubernetes подключает сетевой плагин. |
| Calico | Calico | Сетевой плагин Kubernetes, который у нас отвечает за Pod-сеть. |
| NodePort | NodePort | Способ открыть сервис Kubernetes на порту ноды. |
| кластерный IP | ClusterIP | Внутренний IP сервиса внутри кластера. |
| DNS внутри кластера | in-cluster DNS | Система имён, которая позволяет Pod-ам обращаться к сервисам по имени. |
| доступность | availability | Состояние, при котором сервис или Pod реально готов принимать трафик. |
| `Ready` | Ready | Состояние, при котором нода или Pod считается готовым к работе. |
| `Running` | Running | Состояние запущенного Pod-а. |

## Как связать тезаурус с нашим проектом

Если ученик видит в проекте:

- `common.sh` — это общая подготовка ноды;
- `master.sh` — это bootstrap master;
- `worker.sh` — это join worker-ноды;
- `finalize-cluster.sh` — это финализация сети и Calico;
- `run-post-bootstrap.ps1` — это host-side финальный сценарий;
- `install-dashboard.sh` — это отдельная установка Dashboard в самом конце;
- `smoke-tests/nginx-smoke.yaml` — это учебный smoke-тест.
